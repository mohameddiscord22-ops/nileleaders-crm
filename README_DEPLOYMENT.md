## 🎯 Deployment Steps

### **Option A: Firebase + Railway (Recommended)**

#### Step 1: Deploy Backend on Railway
```bash
# 1. Push your code to GitHub
git push origin main

# 2. Go to railway.app
# 3. Create new project → Deploy from GitHub
# 4. Select your repository
# 5. Add MySQL service
# 6. Set environment variables:
#    - DATABASE_URL (auto-generated by Railway)
#    - JWT_SECRET (generate: openssl rand -base64 32)
#    - SESSION_SECRET (same as JWT_SECRET)
#    - NODE_ENV=production

# 7. Railway auto-deploys on git push
# 8. Get your backend URL from Railway dashboard
```

#### Step 2: Deploy Frontend on Firebase
```bash
# 1. Install Firebase CLI
npm install -g firebase-tools

# 2. Login
firebase login

# 3. Build frontend with backend URL
VITE_API_URL=https://your-railway-backend.railway.app pnpm build

# 4. Deploy
firebase deploy --only hosting

# 5. Your app is live at https://your-project.web.app
```

## 🔐 Security Checklist

Before deploying to production:

```bash
# 1. Generate strong JWT secret
openssl rand -base64 32

# 2. Set environment variables in Railway:
#    - JWT_SECRET=<generated-secret>
#    - SESSION_SECRET=<same-secret>
#    - DATABASE_URL=<railway-mysql-url>
#    - NODE_ENV=production

# 3. Enable HTTPS (automatic on Firebase)

# 4. Test authentication
#    - Sign up with username/password
#    - Verify session cookie is set
#    - Test API calls work

# 5. Verify database backups are enabled
```

## 🧪 Testing Before Deployment

### Local Development
```bash
# 1. Install dependencies
pnpm install

# 2. Set up local .env file
cp .env.example .env
# Edit .env with your local database URL

# 3. Run migrations
pnpm db:push

# 4. Start development server
pnpm dev

# 5. Open http://localhost:3000
```

### Test Authentication
1. Go to login page
2. Create new account (username/password)
3. Verify you're logged in
4. Check browser cookies (should see `session` cookie)
5. Refresh page - should stay logged in

### Test API Connection
1. Open browser DevTools → Network
2. Make an API call (e.g., fetch leads)
3. Verify request goes to correct backend URL
4. Check response status is 200

## 🚨 Common Issues & Solutions

### Issue: `CORS Error` when calling API
**Solution**: 
- Verify `VITE_API_URL` is set during build
- Check backend CORS is configured
- Ensure credentials are included in requests

### Issue: `401 Unauthorized` on API calls
**Solution**:
- Verify `JWT_SECRET` is set on backend
- Check session cookie is being sent
- Verify `SESSION_SECRET` matches `JWT_SECRET`

### Issue: Database connection fails
**Solution**:
- Verify `DATABASE_URL` format is correct
- Check MySQL is accessible
- Test connection manually

### Issue: Firebase deploy fails
**Solution**:
- Run `pnpm build` locally first
- Verify `dist/public` exists
- Check `firebase.json` is correct

## 🔄 Continuous Deployment

### Automatic Updates
1. Push code to GitHub main branch
2. Railway automatically rebuilds and deploys backend
3. Firebase needs manual redeploy (or set up GitHub Actions)

### Manual Redeploy
```bash
# Frontend
VITE_API_URL=https://your-backend.railway.app pnpm build
firebase deploy --only hosting

# Backend (automatic on git push to Railway)
git push origin main
```
